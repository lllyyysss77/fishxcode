# FishXCode 退款政策

**生效日期：2026年4月22日**

本政策适用于 FishXCode 平台内的充值余额退款、Codex 套餐折算，以及 Claude 套餐的特殊售后说明。若本政策与其他页面存在不一致，以本政策为准。

## 一、适用范围

- **充值余额退款**：适用于用户主动充值形成的账户余额。
- **Codex 套餐折算**：适用于仍在有效期内、且满足折算条件的 Codex 套餐。
- **Claude 套餐售后**：适用于 Claude 天卡、月卡等特殊套餐的退款与补偿说明。

## 二、充值余额退款

- 用户主动充值形成的账户余额，支持按 **1:1** 退回至原支付方式。
- 退款过程中产生的支付通道手续费、银行手续费或其他相关费用，由用户自行承担。
- 原路退款的到账时间，以对应支付渠道、银行或支付机构的实际处理时效为准。
- 如遇支付渠道限制、账户异常或风控审核，平台可能要求用户补充身份或订单信息后处理。

## 三、Codex 套餐退款与折算

对符合条件的 Codex 套餐，平台将按照以下规则计算可返还额度，并支持退回账户余额或原支付方式。具体退款路径以系统实际支持情况、支付渠道能力及审核结果为准。

### 1. 计算公式

**可退金额 = 套餐折算基价 -（从计费生效至申请退款时的总计 Token 消耗 × 按量消耗比例）**

其中：

- **套餐折算基价**：以对应支付订单的实付金额为准
- **总计 Token 消耗**：指从套餐计费生效开始至提交退款申请时，套餐项下实际发生的累计 Token 消耗
- **按量消耗比例**：指申请退款时系统对对应模型、分组或服务生效的按量计费换算标准
- 若计算结果小于 **0**，则按 **0** 处理
- 如退回账户余额，最终结果将按照系统当时生效的额度换算规则转换为账户余额

### 2. 折算规则

- 仅处理当前仍在有效期内且命中活动范围的套餐；已过期、已作废、未命中的套餐不会进入折算。
- 只有能关联到历史成功支付订单的购买套餐才可申请折算；赠送、兑换、后台补发等没有支付订单的套餐不参与折算。
- 每个可折算套餐都会展示对应支付订单的实付价格，折算基价以该订单实付金额为准。
- Codex 套餐的退款与折算，基于套餐实际使用情况按量核算，不再按固定周期天数平均折算。
- 平台将以申请退款时可核验的累计 Token 消耗记录、当时生效的按量计费标准及相关系统规则，综合核定可退金额。
- 鉴于计费价格可能随市场与服务成本变化而调整，Codex 套餐相关的退款、折算申请或费用争议，平台可结合申请时的市场价格与系统规则进行处理。

### 3. 提交与审核规则

- 提交申请后，命中的旧套餐会立即暂时禁用，等待管理员审核期间将无法继续使用这些套餐权益。
- 审核通过后，系统将按审核结果退回账户余额或原支付方式，并将这些旧套餐正式作废；审核拒绝则恢复原套餐的可用状态。
- 如退回账户余额，该部分余额仍按您当前系统适用的标准钱包计费规则扣减，与普通充值余额一致。
- 该操作提交后不可自行撤销；如您仍希望继续使用原套餐权益，请勿提交申请。

## 四、Claude 套餐特别说明

- Claude 天卡、月卡套餐因服务性质特殊，**不支持退款**。
- 若某一自然日内，套餐实际稳定可用次数低于该套餐承诺次数的 **80%**，平台可按售后核查结果给予 **补偿 1 天对应次数额度**。
- 前述补偿仅适用于稳定性不足场景，不等同于退款，不折现，亦不退回原支付方式。

## 五、不支持退款或折算的情形

- 已过期、已作废或不在活动范围内的套餐。
- 无法关联到成功支付订单的套餐或权益。
- 赠送、兑换、后台补发、活动赠送等非实际支付取得的权益。
- 已明确标注为不可退款的特殊套餐或特殊活动商品。
- 因用户账户异常、欺诈风险、违规使用等情形被限制处理的订单。

## 六、申请与处理说明

- 用户应通过平台提供的退款、折算或售后入口提交申请，并以系统展示的可退范围、可折算范围及审核结果为准。
- 平台有权基于订单状态、支付记录、套餐状态、风控结果进行人工或系统审核。
- 如用户对审核结果存在异议，可联系平台客服并提供订单号、支付记录和相关说明进行复核。

## 七、政策更新

FishXCode 可根据业务规则、支付通道要求或法律法规更新本政策。更新后的版本将在平台公示，并自公示或标注生效日期起生效。
